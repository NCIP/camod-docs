The file name is caMOD_release_2_4.eap

 

Some of the changes between caMOD 2.1 and caMOD 2.4 are:

-          all definitions have been moved to the tagged values

-          all attributes which were previously labeled “UnctrlVocab” have been renamed to “AlternEntry” and the attributes have been defined as
 “<NAME OF THE ATTRIBUTE FOR WHICH THIS IS AN ALTERNATE ENTRY>.  Free text entered alternatively when the desired option / term is not available from provided vocabulary”
Example: “Environmental Factor name.  Free text entered alternatively when the desired option / term is not available from provided vocabulary”

-          Graft object has been renamed to “Transplantation” and is now defined as “Transference of a tissue or organ, alive or dead, within an individual, between individuals of the same species, or between individuals of different species.”
One of the recommendation was to separate this in subclasses for allograft and xenograft. We believe that this solution is more global, because it covers all possible scenarios including autografts, transplantation of cell lines as well as fresh tissue samples.

 

Please let me know if these changes are acceptable.

 

The next problems to tackle are the inclusion of standards and the enumerating of values. I will review the existing standards and consult you on how to integrate them in caMOD.
